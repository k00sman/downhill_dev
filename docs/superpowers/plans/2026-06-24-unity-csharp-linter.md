# Unity C# Linter Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Give the Downhill repo a cross-platform linter that auto-fixes formatting/style and reports+blocks on code-quality issues for gameplay C# only.

**Architecture:** A self-contained SDK-style "lint project" (`tools/lint/Downhill.Lint.csproj`) compiles only the in-scope gameplay source via `<Compile>` globs and references the Unity engine DLLs + `Library/ScriptAssemblies/*.dll` + NUnit directly — avoiding Unity's stale, transitively-broken generated csproj. Paired `scripts/lint.{sh,ps1}` harvest the one machine-specific path (the engine managed dir) into a git-ignored `tools/lint/Local.props`, then run Phase 1 (`dotnet format`, auto-fix) and Phase 2 (`dotnet build` with analyzers, report+block). An opt-in pre-commit hook runs the same.

**Tech Stack:** .NET SDK 10.x, `dotnet format`, `dotnet build`, `Microsoft.Unity.Analyzers` 1.26.0, built-in .NET analyzers, `.editorconfig`, bash + PowerShell.

## Global Constraints

- **Spec:** `docs/superpowers/specs/2026-06-23-unity-csharp-linter-design.md` (authoritative; mechanism validated by spike).
- **Unity:** 6000.3.17f1 — lint project must use `TargetFramework=netstandard2.1`, `LangVersion=9.0` to match Unity's compile settings.
- **Analyzer version:** `Microsoft.Unity.Analyzers` **1.26.0** exactly.
- **Scope is structural:** compile globs cover **only** `Assets/Scripts/`, `Assets/Editor/`, `Assets/Tests/`. Never add vendor folders (`Assets/Retro Shaders Pro/`, `Assets/Adrift Team/`, `Assets/TutorialInfo/`, `Assets/Fonts/`, PathCreator, etc.).
- **Cross-platform:** every script ships as paired `.sh` (bash) + `.ps1` (PowerShell) in `scripts/`, per AGENTS.md.
- **Prerequisite for running the linter:** the project has been opened/compiled in Unity at least once (so `Library/ScriptAssemblies/*.dll` and at least one root `*.csproj` exist).
- **`.gitignore` already un-ignores `tools/**/*.csproj`** — the lint csproj is committed; `tools/lint/Local.props`, `tools/lint/obj/`, `tools/lint/bin/` must be ignored.
- **Severity split:** formatting/style auto-fixed; code-quality findings reported via build, error-severity blocks.
- **No `.meta` files needed** — `tools/` is outside `Assets/`, so Unity ignores it.
- **Do not commit unless the user asks** (user global instruction). Each task's "Commit" step is to be run only with the user's go-ahead; agentic workers should pause for it.
- **Cannot run Unity tests headless** — after the baseline format pass, ask the user to run the Unity Test Runner (per AGENTS.md).

### Task 1: Lint project that compiles all in-scope source

**Files:**
- Create: `tools/lint/Downhill.Lint.csproj`

**Interfaces:**
- Produces: a buildable MSBuild project at `tools/lint/Downhill.Lint.csproj` that imports `tools/lint/Local.props` for the `$(UnityManagedDir)` property and compiles `Assets/{Scripts,Editor,Tests}/**/*.cs`.
- Consumes: nothing (first task).

- [ ] **Step 1: Create the lint project**

Create `tools/lint/Downhill.Lint.csproj`:

```xml
<Project Sdk="Microsoft.NET.Sdk">

  <!-- Machine-specific paths (UnityManagedDir). Regenerated by scripts/lint.* each run. -->
  <Import Project="Local.props" Condition="Exists('Local.props')" />

  <PropertyGroup>
    <TargetFramework>netstandard2.1</TargetFramework>
    <LangVersion>9.0</LangVersion>
    <Nullable>disable</Nullable>
    <OutputType>Library</OutputType>
    <EnableDefaultCompileItems>false</EnableDefaultCompileItems>
    <GenerateAssemblyInfo>false</GenerateAssemblyInfo>
    <!-- This is a lint-only project; never produced for runtime. -->
    <EnforceCodeStyleInBuild>true</EnforceCodeStyleInBuild>
    <RunAnalyzersDuringBuild>true</RunAnalyzersDuringBuild>
    <!-- Editor-side define set: matches Assets/Editor + Tests. Runtime-only
         (#else) branches are not analyzed; this is an accepted limitation. -->
    <DefineConstants>UNITY_6000_3_OR_NEWER;UNITY_EDITOR;ENABLE_INPUT_SYSTEM;UNITY_INCLUDE_TESTS;DEBUG;TRACE</DefineConstants>
    <!-- Unity manages all references explicitly; keep MSBuild from adding its own. -->
    <DisableImplicitFrameworkReferences>false</DisableImplicitFrameworkReferences>
  </PropertyGroup>

  <ItemGroup>
    <PackageReference Include="Microsoft.Unity.Analyzers" Version="1.26.0" PrivateAssets="all" />
  </ItemGroup>

  <!-- SCOPE: in-scope gameplay source only. Vendor folders are never included. -->
  <ItemGroup>
    <Compile Include="../../Assets/Scripts/**/*.cs" />
    <Compile Include="../../Assets/Editor/**/*.cs" />
    <Compile Include="../../Assets/Tests/**/*.cs" />
  </ItemGroup>

  <!-- References: engine DLLs (machine-specific dir) + package/user DLLs (repo-relative) + NUnit. -->
  <ItemGroup>
    <_LintRef Include="$(UnityManagedDir)/UnityEngine*.dll" />
    <_LintRef Include="$(UnityManagedDir)/UnityEditor*.dll" />
    <_LintRef Include="../../Library/ScriptAssemblies/*.dll" />
    <_LintRef Include="../../Library/PackageCache/com.unity.ext.nunit*/**/nunit.framework.dll" />
    <Reference Include="@(_LintRef)">
      <Private>false</Private>
    </Reference>
  </ItemGroup>

</Project>
```

**Files:**

**Interfaces:**

- Produces: a repo-root `.editorconfig` defining formatting (4-space, Allman, block namespaces, brace-less-if allowed), and analyzer severities (error/warning/disabled tiers).

- [ ] **Step 1: Create `.editorconfig`**

Create `.editorconfig` at the repo root:

```ini
# Downhill — formatting + analyzer policy. Encodes the conventions already used
# in Assets/Scripts and the linter severity tiers (see linter design spec).
root = true

[*.cs]
indent_style = space
indent_size = 4
end_of_line = lf
insert_final_newline = true
charset = utf-8
trim_trailing_whitespace = true

# --- Layout (matches existing gameplay code) ---
csharp_new_line_before_open_brace = all
csharp_style_namespace_declarations = block_scoped:warning
dotnet_sort_system_directives_first = true
dotnet_separate_import_directive_groups = false

# Brace-less single-line statements are intentional (e.g. `if (x) return;`)
csharp_prefer_braces = when_multiline:silent

# Unity convention: public serialized fields are camelCase. We do NOT define a
# naming rule forcing PascalCase on fields, and we silence the "visible instance
# field" rule so serialized public fields are not flagged.
dotnet_diagnostic.CA1051.severity = none

# Unity assigns serialized fields via the Inspector; never-assigned warnings off.
# (Microsoft.Unity.Analyzers also suppresses these for [SerializeField]/MonoBehaviour.)
dotnet_diagnostic.CS0649.severity = none
dotnet_diagnostic.CS0169.severity = none

# === Severity tiers (linter Phase 2 gate) =================================
# Default: keep general analyzer noise as suggestions/warnings, not errors.
dotnet_analyzer_diagnostic.severity = warning

# --- ERROR (blocks the commit hook) ---
# Unity-specific suspicious patterns:
dotnet_diagnostic.UNT0001.severity = error   # empty Unity message method
dotnet_diagnostic.UNT0007.severity = error   # null-coalescing on UnityEngine.Object
dotnet_diagnostic.UNT0008.severity = error   # null-propagation on UnityEngine.Object
# Code-quality:
dotnet_diagnostic.CS0219.severity = error    # variable assigned but never used
dotnet_diagnostic.IDE0059.severity = error   # unnecessary value assignment
dotnet_diagnostic.IDE0051.severity = error   # unused private member
dotnet_diagnostic.CS0162.severity = error    # unreachable code

# --- DISABLED (misfire in Unity / stylistic noise we don't want to block) ---
dotnet_diagnostic.IDE0058.severity = none    # expression value never used (common in Unity APIs)
```

### Task 3: `scripts/lint.sh` — harvest, format, build

**Files:**
- Create: `scripts/lint.sh`

**Interfaces:**
- Consumes: `tools/lint/Downhill.Lint.csproj` (Task 1), `.editorconfig` (Task 2).
- Produces: `scripts/lint.sh` — runs `[--check]`; regenerates `tools/lint/Local.props`, runs Phase 1 (`dotnet format`) then Phase 2 (`dotnet build`); exit code 0 = clean, non-zero = needs formatting (in `--check`) or has error-tier findings.

- [ ] **Step 1: Create `scripts/lint.sh`**

```bash
#!/usr/bin/env bash
set -euo pipefail

REPO=$(git rev-parse --show-toplevel)
LINT_DIR="$REPO/tools/lint"
PROJ="$LINT_DIR/Downhill.Lint.csproj"

command -v dotnet >/dev/null 2>&1 || {
  echo "error: 'dotnet' not found. Install the .NET SDK 10.x." >&2; exit 1; }

# --- Harvest the one machine-specific path: the Unity engine managed dir. ---
CSPROJ=$(ls "$REPO"/*.csproj 2>/dev/null | head -1 || true)
if [ -z "${CSPROJ:-}" ]; then
  echo "error: no Unity-generated .csproj found at repo root." >&2
  echo "       Open the project in Unity once to generate them." >&2
  exit 1
fi
HINT=$(grep -m1 -oE '<HintPath>[^<]*UnityEngine/UnityEngine\.dll</HintPath>' "$CSPROJ" \
        | sed -E 's#</?HintPath>##g' || true)
if [ -z "${HINT:-}" ]; then
  echo "error: could not find the UnityEngine.dll HintPath in $CSPROJ." >&2
  exit 1
fi
MANAGED_DIR=$(dirname "$HINT")
if [ ! -d "$REPO/Library/ScriptAssemblies" ]; then
  echo "error: Library/ScriptAssemblies missing. Let Unity compile the project first." >&2
  exit 1
fi

printf '<Project>\n  <PropertyGroup>\n    <UnityManagedDir>%s</UnityManagedDir>\n  </PropertyGroup>\n</Project>\n' \
  "$MANAGED_DIR" > "$LINT_DIR/Local.props"

CHECK=0
[ "${1:-}" = "--check" ] && CHECK=1

if [ "$CHECK" = "1" ]; then
  echo "==> Phase 1: format (verify only)"
  dotnet format whitespace "$PROJ" --verify-no-changes
  dotnet format style "$PROJ" --verify-no-changes
else
  echo "==> Phase 1: format (auto-fix)"
  dotnet format whitespace "$PROJ"
  dotnet format style "$PROJ"
fi

echo "==> Phase 2: quality build"
dotnet build "$PROJ" -clp:NoSummary -v quiet

echo "lint: OK"
```

- [ ] **Step 1: Create `scripts/lint.ps1`**

```powershell
#Requires -Version 5.1
$ErrorActionPreference = 'Stop'

$repo    = git rev-parse --show-toplevel
$lintDir = Join-Path $repo 'tools/lint'
$proj    = Join-Path $lintDir 'Downhill.Lint.csproj'

if (-not (Get-Command dotnet -ErrorAction SilentlyContinue)) {
    Write-Error "'dotnet' not found. Install the .NET SDK 10.x."; exit 1
}

# Harvest the Unity engine managed dir from any generated csproj.
$csproj = Get-ChildItem -Path $repo -Filter '*.csproj' -File | Select-Object -First 1
if (-not $csproj) {
    Write-Error "No Unity-generated .csproj found at repo root. Open the project in Unity once."; exit 1
}
$m = Select-String -Path $csproj.FullName -Pattern '<HintPath>([^<]*UnityEngine/UnityEngine\.dll)</HintPath>' |
     Select-Object -First 1
if (-not $m) { Write-Error "Could not find UnityEngine.dll HintPath in $($csproj.Name)."; exit 1 }
$hint = $m.Matches[0].Groups[1].Value
$managedDir = Split-Path -Parent $hint
if (-not (Test-Path (Join-Path $repo 'Library/ScriptAssemblies'))) {
    Write-Error "Library/ScriptAssemblies missing. Let Unity compile the project first."; exit 1
}

@"
<Project>
  <PropertyGroup>
    <UnityManagedDir>$managedDir</UnityManagedDir>
  </PropertyGroup>
</Project>
"@ | Set-Content -Path (Join-Path $lintDir 'Local.props') -Encoding UTF8

$check = ($args.Count -ge 1 -and $args[0] -eq '--check')

if ($check) {
    Write-Host "==> Phase 1: format (verify only)"
    dotnet format whitespace $proj --verify-no-changes
    dotnet format style $proj --verify-no-changes
} else {
    Write-Host "==> Phase 1: format (auto-fix)"
    dotnet format whitespace $proj
    dotnet format style $proj
}

Write-Host "==> Phase 2: quality build"
dotnet build $proj -clp:NoSummary -v quiet

Write-Host "lint: OK"
```

### Task 5: Pre-commit hook + opt-in install + gitignore

**Files:**
- Create: `.githooks/pre-commit`
- Modify: `.gitignore` (ignore `tools/lint/Local.props`, `tools/lint/obj/`, `tools/lint/bin/`)

**Interfaces:**
- Consumes: `scripts/lint.sh` (Task 3).
- Produces: `.githooks/pre-commit` that auto-fixes formatting on the working tree, re-stages in-scope files, and blocks the commit when Phase 2 finds error-tier issues. Enabled opt-in via `git config core.hooksPath .githooks`.

- [ ] **Step 1: Create `.githooks/pre-commit`**

```bash
#!/usr/bin/env bash
# Downhill pre-commit linter. Enable with: git config core.hooksPath .githooks
# Auto-fixes formatting (re-staged), then blocks on code-quality errors.
# Bypass intentionally with: git commit --no-verify
set -uo pipefail

REPO=$(git rev-parse --show-toplevel)

# Only act if in-scope files are staged.
if git diff --cached --name-only | grep -qE '^Assets/(Scripts|Editor|Tests)/.*\.cs$'; then
  "$REPO/scripts/lint.sh"
  rc=$?
  # Re-stage any in-scope files the formatter rewrote.
  git add -- "$REPO/Assets/Scripts" "$REPO/Assets/Editor" "$REPO/Assets/Tests" 2>/dev/null || true
  if [ "$rc" -ne 0 ]; then
    echo "pre-commit: lint failed — fix the errors above or use 'git commit --no-verify' to bypass." >&2
    exit "$rc"
  fi
fi
exit 0
```
